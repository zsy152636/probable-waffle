# AI 智能答题系统 — 使用文档

## 一、环境准备与安装

### 1.1 环境要求

- **JDK 17** 或更高版本
- **Node.js 18+**（推荐 20+）
- **pnpm**（推荐包管理器，也可使用 npm）
- **MySQL 8.0+**
- **Redis 7.x**

### 1.2 数据库初始化

```bash
# 步骤 1：创建数据库
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS ai_quiz DEFAULT CHARSET utf8mb4 COLLATE utf8mb4_0900_ai_ci;"

# 步骤 2：修改数据库连接配置
# 打开 ai-quiz/ai-quiz-admin/src/main/resources/application-dev.yml
# 将 spring.datasource.username 和 spring.datasource.password 改为你的 MySQL 账号密码
```

数据库配置示例（`application-dev.yml`）：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/ai_quiz?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai
    username: root           # 改为你的数据库用户名
    password: root         # 改为你的数据库密码
  data:
    redis:
      host: localhost
      port: 6379
      password:              # Redis 密码（无密码则留空）
```

> **说明**：项目启动时会自动执行 `schema.sql` 和 `data.sql` 脚本，初始化表结构和默认数据（含一个管理员账号）。

### 1.3 启动后端

```bash
# 确保 Redis 已启动（默认端口 6379）

# 编译后端
cd ai-quiz
mvn clean install -DskipTests

# 启动后端服务（端口 9010）
mvn spring-boot:run -pl ai-quiz-admin
```

启动成功后控制台会打印 `Started QuizApplication in x.xxx seconds`，同时自动建表并插入初始数据。

### 1.4 启动前端

```bash
cd ai-quiz-web

# 安装依赖
pnpm install

# 启动开发服务器（端口 9020）
pnpm dev

# 如需生产构建
pnpm build
```

打开浏览器访问 `http://localhost:9020` 即可进入登录页面。

### 1.5 默认管理员账号

| 用户名 | 密码 | 角色 |
|-------|------|------|
| `admin` | `admin123` | ADMIN（超级管理员，拥有全部权限） |

> **安全提示**：首次登录后请立即修改默认密码。

## 二、系统功能使用指南

### 2.1 登录

访问 `http://localhost:9020`，进入登录页面：

- 输入用户名和密码
- 点击「登录」按钮
- 登录成功后自动跳转至主页仪表盘
- 如果已登录但 Token 过期，系统会自动跳转回登录页

---

### 2.2 主页仪表盘

主页展示系统的核心统计数据：

**统计卡片（带动画计数）：**
- 文档总数
- 试题总数
- 答题次数
- 正确率（百分比）

**图表分析：**
- **答题正确率** — 饼图展示正确/错误占比，中心显示正确率百分比
- **分数段分布** — 柱状图展示各分数段的答题人数分布（不同颜色区分）
- **答题概况** — 累计做题数、累计正确数、平均得分、平均用时
- **近期得分趋势** — 折线图展示近期每次答题的得分变化趋势（含满分参考线）

**快捷入口：**
- 上传文档 — 跳转到文档列表页
- 试题列表 — 跳转到试题列表页
- 答题记录 — 跳转到答题记录页
- AI 配置 — 跳转到 AI 服务配置页

---

### 2.3 AI 服务配置

> 导航路径：左侧菜单「系统管理 → AI 配置」

**功能：** 在 AI 生成题目前，需要先配置可用的 AI 服务。

**支持的大模型厂商：**
- OpenAI（GPT 系列）
- Anthropic（Claude 系列）
- 任何兼容 OpenAI API 格式的服务（如 DeepSeek、通义千问、文心一言等）

**操作步骤：**

1. 点击「新增配置」按钮
2. 填写配置信息：
   - **厂商**：如 OpenAI、Anthropic、DeepSeek 等（仅用于标识）
   - **API 地址**：AI 服务的 API 端点 URL
   - **API Token**：调用 AI 服务的密钥
   - **模型名称**：如 `gpt-4o`、`claude-sonnet-4-20250514`、`deepseek-chat` 等
   - **Max Tokens**：最大输出 Token 数（默认 4096）
   - **Temperature**：温度参数（默认 0.7，范围 0~2）
3. 点击「新增」保存配置
4. 新增后需「启用」该配置才能用于出题
5. 可使用「测试连接」按钮验证配置是否正确连通

> **注意**：一次只能启用一个 AI 配置。切换启用时会自动禁用其他配置。

---

### 2.4 题目生成配置模板（可选）

> 导航路径：左侧菜单「系统管理 → 题目生成配置」

**功能：** 预设题目生成参数模板，在文档出题时一键应用，免去每次手动配置题型数量的重复操作。

**操作步骤：**

1. 点击「新增配置」按钮
2. 填写配置信息：
   - **配置名称**：如"政考标准配置"、"编程题配置"等
   - **描述**：简要说明此配置的用途
   - **题型明细**：从 7 种题型中选择并填写每种题型的出题数量
     - 单选题（SINGLE）
     - 多选题（MULTI）
     - 判断题（JUDGE）
     - 填空题（FILL）
     - 简答题（SHORT）
     - 代码题（CODE）
     - 计算大题（CALCULATION）
   - **强制阅读材料**：启用后，AI 生成选择题时会附带阅读材料 passage 字段
   - **出题方向**：自由文本描述出题侧重点，如"侧重政策理解与分析"
3. 可创建多个配置模板，但只有「启用」状态的模板可供选择
4. 在文档列表页生成题目时，可从下拉列表选择已启用的配置模板

> **技巧**：可以针对不同科目创建不同的配置模板，如「政治-侧重理解」、「编程-侧重代码实现」等。

---

### 2.5 文档管理

> 导航路径：左侧菜单「应用 → 文档列表」

#### 2.5.1 上传文档

1. 点击「上传文档」按钮
2. 在弹出的上传对话框中，将文件拖拽到上传区域或点击选择文件
3. **支持格式**：TXT（.txt）、Word（.docx）、PDF（.pdf）、Markdown（.md）
4. **单文件限制**：不超过 20MB
5. 支持一次上传多个文件
6. 选择文件后点击「上传」按钮完成上传

#### 2.5.2 文档列表

文档列表以表格形式展示所有已上传的文档，包含以下信息：

| 列名 | 说明 |
|------|------|
| 文档编号 | 唯一编号（WD + 雪花ID） |
| 文档名称 | 文档标题 |
| 类型 | TXT / WORD / PDF |
| 题目进度 | 已确认数 / 总生成数 |
| 题目生成时间 | 最近一次 AI 出题的时间 |
| 上传用户 | 上传者用户名 |
| 上传时间 | 文档上传时间 |

**列表功能：**
- **搜索**：按文档名称关键字搜索
- **类型筛选**：按 TXT / Word / PDF 筛选
- **编辑**：修改文档名称
- **删除**：删除文档（文档被删除后其关联的题目也会被清除）

#### 2.5.3 AI 生成题目

1. 选择一个文档，点击「生成题目」按钮
2. 在弹出的对话框中配置出题参数：
   - **配置模板**：可选择已预置的配置模板（自动填充题型数量）
   - **手动配置**：或手动选择题型并填写每种题型的数量
   - **总题数**：自动汇总显示
   - **强制阅读材料**：是否让 AI 为选择题附带阅读材料
   - **出题方向**：可选填，指导 AI 的出题侧重点
3. 点击「确认生成」提交
4. 系统调用 AI 生成题目，等待返回结果（可能需 30 秒到数分钟，取决于题量）
5. 生成成功后自动跳转到「题目确认」页面

> **注意**：如果该文档已经生成过题目但未确认，再次生成将覆盖原有题目。

---

### 2.6 题目确认

> 导航路径：文档列表 → 点击某个文档的「确认题目」

#### 2.6.1 题目列表

题目按题型分组展示（7 种题型各为一组），包含以下信息：

- 题型标签（不同颜色区分）
- 题干内容
- 阅读材料（如果有）
- 选项列表（正确答案高亮显示）
- 参考答案（填空/简答/代码/计算类题目）
- 答案解析
- 难度（★ 表示）
- 分值
- 确认状态

#### 2.6.2 筛选与搜索

- **状态筛选**：全部 / 待确认 / 已确认
- **题型筛选**：全部 7 种题型可选
- **分页**：支持分页浏览

#### 2.6.3 编辑题目

1. 点击题目卡片上的「编辑」按钮
2. 在弹出的编辑对话框中修改：
   - 题干内容
   - 选项文本及正确答案标记
   - 答案解析
   - 难度（1-简单 / 2-中等 / 3-困难）
   - 分值
3. 点击「保存」提交修改

#### 2.6.4 批量操作

- **全选 / 取消全选**：一键选择当前筛选条件下的全部题目
- **按题型全选**：点击题型标题旁的复选框，选择该题型下的全部题目
- **批量确认**：选择题目后点击「批量确认」，已确认的题目将自动生成试卷
- **批量删除**：选择题目后点击「批量删除」，可删除不需要的题目

#### 2.6.5 重新生成

点击「重新生成」将清空当前文档的所有题目并重新调用 AI 生成，会弹出确认提示。

#### 2.6.6 取消生成

点击「取消生成」清空当前文档的所有题目（待确认状态），保留已确认的题目。

> **注意**：题目一旦确认（status = 1），将自动归入对应试卷，不可再次编辑。

> 导航路径：左侧菜单「应用 → 试题列表」

**功能：** 展示所有已确认生成的试卷。

**列表信息：**

| 列名 | 说明 |
|------|------|

| 试题名称 | 试卷名称（取自文档名） |

### 2.8 在线答题

> 导航路径：试题列表 → 点击「测试」

#### 2.8.1 答题模式

进入答题页面后，系统自动开始计时：

- **顶部信息栏**：
  - 返回试题列表按钮
  - 文档名称
  - 剩余时间倒计时（≤ 5 分钟时变红警告）
  - 答题进度（已答 / 总数）

- **题目展示**：按题型分组显示
- **7 种题型作答方式**：

| 题型 | 操作方式 |
|------|---------|
| 单选题 | 点击 Radio 选择唯一答案 |
| 多选题 | 勾选 Checkbox 选择一个或多个答案 |
| 判断题 | 选择"正确"或"错误" |
| 填空题 | 在文本框中输入答案 |
| 简答题 | 在文本域中输入文字作答 |
| 代码题 | 在等宽字体文本域中编写代码 |
| 计算大题 | 在文本域中输入解题过程和答案 |

#### 2.8.2 提交答案

1. 完成所有题目（或部分作答）
2. 点击页面底部的「提交」按钮
3. 系统自动批改客观题，并展示答题结果

#### 2.8.3 成绩展示

提交后进入成绩页面，展示以下信息：

- **总览**：得分、总分、正确题数、总题数、用时
- **详细结果**：每道题的作答情况
  - 正确答案高亮显示
  - 用户所选答案标记（正确绿色 / 错误红色）
  - 每题得分
  - 答案解析

## 三、用户与权限管理

### 3.1 用户管理

> 导航路径：左侧菜单「系统管理 → 用户管理」

**功能：**
- 查看用户列表（用户名、姓名、邮箱、手机号、状态、创建时间）
- 新增用户（填写用户名、密码、姓名、邮箱、手机号）
- 编辑用户（修改基本信息）
- 启用/禁用用户（禁用后用户无法登录）
- 分配角色（一个用户可拥有多个角色）
- 关键词搜索

### 3.2 角色管理

> 导航路径：左侧菜单「系统管理 → 权限管理」

**功能：**
- 查看角色列表
- 新增/编辑角色（角色名称、角色编码、描述）
- 分配菜单/按钮权限（树形结构选择）
- 删除角色

### 3.3 权限说明

系统使用 RBAC（基于角色的访问控制）模型：

```
用户 ←→ 角色 ←→ 菜单/按钮权限
```

- 一个用户可以拥有多个角色
- 一个角色可以绑定多个菜单和按钮权限
- ADMIN 角色拥有全部权限，不受限制
- 前端根据权限标识控制按钮的显隐，后端接口也有权限拦截

**常用权限标识：**

| 权限标识 | 功能点 |
|---------|--------|
| `user:list` | 查看用户列表 |
| `user:add` | 新增用户 |
| `user:update` | 修改用户 |
| `user:delete` | 删除用户 |
| `role:list` | 查看角色列表 |
| `role:add` | 新增角色 |
| `role:update` | 修改角色 |
| `role:delete` | 删除角色 |
| `ai:list` | 查看 AI 配置 |
| `ai:add` | 新增 AI 配置 |
| `ai:update` | 修改/启用/禁用 AI 配置 |
| `ai:delete` | 删除 AI 配置 |
| `ai:test` | 测试 AI 连接 |
| `doc:list` | 查看文档列表 |
| `doc:upload` | 上传文档 |
| `doc:update` | 修改文档名称 |
| `doc:delete` | 删除文档 |
| `doc:genQuestion` | AI 生成题目 + 管理题目生成配置 |
| `question:confirm` | 确认/编辑题目 |
| `paper:list` | 查看试题列表 |
| `paper:delete` | 删除试题 |
| `quiz:take` | 在线答题 + 查看答题记录 |

## 五、常见问题 FAQ

### Q1：AI 出题失败怎么办？

可能原因：
- AI 配置未启用 → 检查 AI 配置页面，确保有配置处于启用状态
- API Token 错误 → 使用「测试连接」功能验证
- AI 服务不可达 → 检查 API 地址是否正确，网络是否连通
- Token 限额不足 → 检查 AI 账号的 Token 额度
- 文档内容过少 → AI 需要足够的文档内容进行分析，建议文档不少于 500 字

### Q2：试题生成后在哪里查看？

文档列表 → 点击文档的「确认题目」按钮，进入题目确认页面查看 AI 生成的题目。

### Q3：如何修改已确认的题目？

题目确认后（status = 1）不可再编辑。如需修改，需删除该试卷后重新生成。

### Q4：默认管理员密码是多少？

用户名 `admin`，密码 `admin123`。首次登录后建议立即修改密码。

### Q5：如何创建普通用户？

系统管理 → 用户管理 → 点击「新增」→ 填写用户信息 → 分配角色 → 保存。

### Q6：一个文档可以生成多次题目吗？

可以。但重新生成会覆盖该文档下所有待确认的题目（已确认的不受影响）。

### Q7：支持哪些 AI 模型？

支持任何兼容 OpenAI API 格式的服务，包括但不限于：
- OpenAI GPT 系列（gpt-4o、gpt-4-turbo 等）
- Anthropic Claude 系列（通过 API 代理）
- DeepSeek
- 通义千问
- 文心一言
- 智谱 GLM
- 本地部署的 ollama / vLLM 等

### Q8：如何备份数据？

数据库备份：
```bash
mysqldump -u root -p ai_quiz > ai_quiz_backup.sql
```

## 六、开发环境配置参考

### 6.1 前端端口配置

编辑 `ai-quiz-web/vite.config.ts`：


## 七、技术说明

### 7.1 前端技术说明

- **Vue 3 Composition API** + `<script setup>` 语法
- **TypeScript** 全类型覆盖，类型定义集中在 `src/types/index.ts`
- **Element Plus** 中文国际化
- **Pinia** 状态管理（user store + app store）
- **Vue Router** 导航守卫实现登录校验和权限控制
- **Axios** 统一请求拦截（注入 Token）+ 响应拦截（统一错误处理 + 401 自动跳转登录）
- **ECharts** 仪表盘图表（饼图、柱状图、折线图）+ 动画过渡
- **CSS Variables** 主题系统（`theme.css` 定义全局颜色变量）

### 7.2 后端技术说明

- **Spring Boot 3.2.5** + JDK 17
- **MyBatis-Plus** 自动分页、雪花 ID 主键生成、逻辑删除
- **Spring Security** + JWT 无状态认证
- **Redis** 缓存（Spring Data Redis + Lettuce 连接池）
- **Apache POI** 解析 Word 文档
- **Apache PDFBox** 解析 PDF 文档
- **OkHttp** 调用第三方 AI API
- **统一异常处理**（GlobalExceptionHandler + BusinessException）
- **统一返回格式**（Result<T> 结构）

### 7.3 数据库说明

- 主键使用雪花算法（Snowflake ID）生成
- 所有业务表支持逻辑删除（`deleted` 字段）
- 自动填充 `create_time` 和 `update_time`
- 数据库自动初始化：首次启动执行 `schema.sql`（建表）+ `data.sql`（初始数据）
