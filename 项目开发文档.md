# AI 智能答题系统 — 项目开发文档

## 一、项目概述

### 1.1 项目简介

AI 智能答题系统是一套基于大语言模型的在线考试/答题生成平台。用户上传文档（TXT / Word / PDF），系统调用可配置的 AI 服务对文档内容进行语义分析，自动提取知识点并生成结构化题库，最终输出为在线答题页面供考生使用。

### 1.2 核心功能

| 模块 | 功能点 |
|------|--------|
| 用户认证 | 登录 / 登出 / JWT Token 续期 |
| 系统管理 - 用户管理 | 用户 CRUD、角色分配、启用/禁用 |
| 系统管理 - 权限管理 | 角色 CRUD、菜单/按钮级权限绑定 |
| 系统管理 - AI 配置 | 服务厂商、API Token、访问地址自由配置 |
| 应用 - 文档列表 | 文档上传、列表展示、编辑、删除、AI 题目生成（数量/题型可配） |
| 题目确认 | AI 生成题目回显前端，用户确认后自动生成试题并存入试题列表 |
| 应用 - 试题列表 | 已确认的试题展示、删除、点击进入答题页 |
| 在线答题 | 试题列表中点击"测试"进入答题页，提交后自动评分并展示成绩 |

### 1.3 技术选型

| 层级 | 技术 | 版本建议 |
|------|------|---------|
| 后端框架 | Spring Boot | 3.2.x |

### 2.3 模块依赖关系

```
                    ┌──────────────┐
                    │ ai-quiz-admin │  ← 启动模块，聚合所有模块
                    └──────┬───────┘
                ┌──────────┼──────────┐
                │          │          │
        ┌───────▼──┐ ┌────▼────┐ ┌───▼────────┐
        │ai-quiz- │ │ai-quiz- │ │ai-quiz-sys  │
        │  dome   │ │  ai     │ │ (基础模块)   │
        └───┬──┬──┘ └───┬─────┘ └─────────────┘
            │  │        │            ▲
            │  └────────┤            │
            │           │            │
            └───────────┴────────────┘
          (dome 依赖 ai 调用 AI 生成题目)
          (dome 依赖 sys 获取公共类/鉴权)
          (ai   依赖 sys 获取公共类)
          (admin 依赖全部模块)
```

| 依赖方向 | 说明 |
|----------|------|
| `ai-quiz-sys` → 无 | 基础模块，不依赖其他业务模块 |
| `ai-quiz-ai` → `ai-quiz-sys` | AI 模块引用公共类（Result、BusinessException 等） |
| `ai-quiz-dome` → `ai-quiz-sys` | 业务模块引用公共类、鉴权拦截器 |
| `ai-quiz-dome` → `ai-quiz-ai` | 文档生成题目时调用 AIService |
| `ai-quiz-admin` → 全部 | 启动模块聚合 sys、ai、dome，引入所有依赖并装配 |

### 2.4 模块职责划分

| 模块 | 核心职责 | 包含内容 |
|------|---------|---------|
| **ai-quiz-sys** | 系统基础设施 | 认证鉴权（Spring Security + JWT）、用户管理、角色权限管理、统一返回体、分页工具、雪花 ID、全局异常处理、Redis/MyBatis-Plus 配置 |
| **ai-quiz-ai** | AI 大模型交互 | AI 配置管理（厂商/Token/模型）、OkHttp 动态客户端、Prompt 模板引擎、AI 返回 JSON 解析、连通性测试 |
| **ai-quiz-dome** | 领域业务逻辑 | 文档上传解析、题目 CRUD + 确认、试题发布、在线答题、交卷评分、简答题 AI 评分 |
| **ai-quiz-admin** | 应用启动入口 | Spring Boot 主类、`application.yml` 配置聚合、DB 初始化脚本、`@ComponentScan` 跨模块扫描 |

### 3.2 建表 SQL

```sql
-- ===================== 用户 & 权限 =====================

CREATE TABLE `sys_user` (
    `id`            BIGINT       NOT NULL COMMENT '主键（雪花ID）',
    `username`      VARCHAR(50)  NOT NULL COMMENT '用户名',
    `password`      VARCHAR(255) NOT NULL COMMENT '密码（BCrypt）',
    `real_name`     VARCHAR(50)  DEFAULT NULL COMMENT '真实姓名',
    `email`         VARCHAR(100) DEFAULT NULL COMMENT '邮箱',
    `phone`         VARCHAR(20)  DEFAULT NULL COMMENT '手机号',
    `status`        TINYINT      NOT NULL DEFAULT 1 COMMENT '状态：0-禁用 1-启用',
    `create_time`   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `update_time`   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    `deleted`       TINYINT      NOT NULL DEFAULT 0 COMMENT '逻辑删除：0-未删除 1-已删除',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_username` (`username`, `deleted`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统用户';

CREATE TABLE `sys_role` (
    `id`            BIGINT       NOT NULL COMMENT '主键（雪花ID）',
    `role_name`     VARCHAR(50)  NOT NULL COMMENT '角色名称',
    `role_code`     VARCHAR(50)  NOT NULL COMMENT '角色编码',
    `description`   VARCHAR(200) DEFAULT NULL,
    `create_time`   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `update_time`   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    `deleted`       TINYINT      NOT NULL DEFAULT 0,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_role_code` (`role_code`, `deleted`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='角色';

CREATE TABLE `sys_menu` (
    `id`            BIGINT       NOT NULL COMMENT '主键',
    `parent_id`     BIGINT       NOT NULL DEFAULT 0 COMMENT '父菜单ID',
    `menu_name`     VARCHAR(50)  NOT NULL COMMENT '菜单名称',
    `path`          VARCHAR(200) DEFAULT NULL COMMENT '路由路径',

### 4.7 题目生成 & 确认模块

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST   | `/api/documents/{docId}/generate-questions` | 触发 AI 题目生成（传入数量 & 题型配置） | `doc:genQuestion` |
| GET    | `/api/documents/{docId}/questions?status=` | 获取题目列表（可按状态筛选） | `doc:list` |
| PUT    | `/api/questions/{id}` | 编辑单道题目 | `question:confirm` |
| DELETE | `/api/questions/{id}` | 删除单道题目 | `question:confirm` |
| POST   | `/api/questions/batch-confirm` | 批量确认题目，**同时自动生成试题记录** | `question:confirm` |
| POST   | `/api/questions/batch-delete` | 批量删除题目 | `question:confirm` |

**题目生成请求体 `POST /api/documents/{docId}/generate-questions`：**

```json
{
  "totalCount": 20,
  "includeFill": false,
  "typeRatios": {
    "single": 50,
    "multi": 30,
    "judge": 20
  }
}
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `totalCount` | int | 总题数，可选值：20 / 50 / 60 |
| `includeFill` | boolean | 是否包含填空题，默认 false |
| `typeRatios.single` | int | 单选题占比(%) |
| `typeRatios.multi` | int | 多选题占比(%) |
| `typeRatios.judge` | int | 判断题占比(%) |

> 各占比之和固定为 100%。若 `includeFill = true`，则填空题从 totalCount 中额外分配（从各题型中等比例扣减，填空题固定占 10%）。
>
> **前端生成对话框**：点击"题目生成"按钮 → 弹出 `<el-dialog>` → 下拉选择总题数（20/50/60）、复选框是否包含填空题、拖拽滑块或输入框调整单选/多选/判断比例（默认 50:30:20）→ 确认后发起请求。

**评分规则（后端实现）：**

| 题型 | 计分方式 |
|------|---------|
| 单选题 SINGLE | 所选选项与正确答案完全一致即满分，否则 0 分 |
| 多选题 MULTI | 全部选对且无多选/漏选即满分，部分正确不得分 |
| 判断题 JUDGE | 与正确答案一致即满分 |
| 填空题 FILL | 用户答案与正确答案字符串模糊匹配（忽略大小写、前后空格） |
| 简答题 SHORT | AI 辅助评分（调用 AI 对比用户答案与参考答案的语义相似度），0 到满分 |

| 字段 | 类型 | 说明 |
|------|------|------|
| `confirmedCount` | int | 已确认题目数 |
| `hasPaper` | boolean | 是否已生成试题（`confirmedCount === questionCount && questionCount > 0`） |
| `paperId` | string | 试题ID（`hasPaper=true` 时返回，用于跳转试题详情） |

> 前端按钮逻辑：
> - `questionCount === 0` → 【题目生成】
> - `0 < confirmedCount < questionCount` → 【查看题目】
> - `confirmedCount === questionCount && questionCount > 0` → 显示"试题已生成"标签，操作列不含测试入口（测试入口在试题列表页）

### 4.11 AI 题目生成 — 核心流程

**触发生成 `POST /api/documents/{docId}/generate-questions`**

1. 后端根据 `docId` 查询文档，获取已解析的纯文本 `content_text`
2. 查询当前激活的 AI 配置（`is_active = 1`）
3. 根据请求参数动态构建 System Prompt（以下以 totalCount=20, 单选50%/多选30%/判断20%, 不含填空为例）：

```
你是一个专业的考试出题专家。请根据以下文档内容，生成一套高质量的题目。

【题目要求】
- 总题数：20 道
- 题型分布：
  单选题(SINGLE)：10 道（50%）
  多选题(MULTI)：6 道（30%）
  判断题(JUDGE)：4 道（20%）
- 填空题为可选，本次不包含

【出题规范】
1. 题目覆盖文档的核心知识点，避免重复考点
2. 每道题标注难度：1-简单 2-中等 3-困难，三种难度均匀分布
3. 每道题包含答案解析
4. 单选题和多选题均需4个选项(A/B/C/D)，并标注正确答案
5. 判断题选项固定为"正确"/"错误"
6. 每道题默认分值 5 分
7. 返回严格的 JSON 格式，不要包含任何其他文字

返回 JSON 格式如下：
{
  "questions": [
    {
      "questionType": "SINGLE",
      "questionTitle": "Java中int的取值范围是？",
      "options": [
        { "label": "A", "text": "-128 到 127", "isCorrect": false },
        { "label": "B", "text": "-2^31 到 2^31-1", "isCorrect": true },
        { "label": "C", "text": "0 到 65535", "isCorrect": false },
        { "label": "D", "text": "-2^63 到 2^63-1", "isCorrect": false }
      ],
      "analysis": "int是32位有符号整数",
      "difficulty": 1,
      "score": 5
    },
    {
      "questionType": "JUDGE",
      "questionTitle": "Java中String是基本数据类型。",
      "options": [
        { "label": "A", "text": "正确", "isCorrect": false },
        { "label": "B", "text": "错误", "isCorrect": true }
      ],
      "analysis": "String是引用类型，Java基本数据类型只有8种。",
      "difficulty": 1,
      "score": 5
    },
    ...
  ]
}
```

4. **AIService 动态构建 Prompt**：根据 `totalCount`、`includeFill`、`typeRatios` 参数计算各题型数量，拼入 System Prompt
5. 调用 AI API，传入文档内容 + Prompt
6. 解析 AI 返回的 JSON
7. 批量保存题目到数据库（status = 0 待确认）
8. 更新文档的 `question_gen_time` 和 `question_count`
9. 返回生成的题目列表给前端

**前端确认页功能：**
- 展示所有未确认题目（按题型分组）
- 每道题可编辑（题干、选项、正确答案、解析、难度、分值）
- 支持单题删除
- 支持批量确认 / 批量删除
- **确认后自动创建试题**：`POST /api/questions/batch-confirm` 后端同时插入 `test_paper` 记录并返回 `paperId`，前端可引导用户跳转试题列表

### 7.1 DocumentParser 工具类设计

```java
// 伪代码结构
public class DocumentParser {

    public static String parseText(MultipartFile file) {
        String ext = getExtension(file.getOriginalFilename());
        return switch (ext) {
            case "txt"  -> new String(file.getBytes(), StandardCharsets.UTF_8);
            case "docx" -> parseWord(file.getInputStream());
            case "pdf"  -> parsePdf(file.getInputStream());
            default     -> throw new BusinessException("不支持的文件类型");
        };
    }

    private static String parseWord(InputStream is) {
        // Apache POI XWPFDocument
        XWPFDocument doc = new XWPFDocument(is);
        StringBuilder sb = new StringBuilder();
        for (XWPFParagraph p : doc.getParagraphs()) {
            sb.append(p.getText()).append("\n");
        }
        return sb.toString();
    }

    private static String parsePdf(InputStream is) {
        // Apache PDFBox
        PDDocument doc = PDDocument.load(is);
        PDFTextStripper stripper = new PDFTextStripper();
        String text = stripper.getText(doc);
        doc.close();
        return text;
    }
}
```

## 八、AI 调用核心代码设计

### 8.1 AIClientFactory — 动态构建客户端

```java
@Component
public class AIClientFactory {

    private final OkHttpClient httpClient = new OkHttpClient.Builder()
            .connectTimeout(30, TimeUnit.SECONDS)
            .readTimeout(120, TimeUnit.SECONDS)  // AI 返回可能较慢
            .build();

    private final ObjectMapper objectMapper = new ObjectMapper();

    /**
     * 根据数据库配置调用 AI 服务
     */
    public String callAI(AIConfig config, String systemPrompt, String userMessage) {
        Map<String, Object> body = Map.of(
            "model", config.getModelName(),
            "messages", List.of(
                Map.of("role", "system", "content", systemPrompt),
                Map.of("role", "user", "content", userMessage)
            ),
            "max_tokens", config.getMaxTokens(),
            "temperature", config.getTemperature()
        );

        Request request = new Request.Builder()
                .url(config.getApiUrl())
                .post(RequestBody.create(
                    objectMapper.writeValueAsString(body),
                    MediaType.parse("application/json")))
                .header("Authorization", "Bearer " + config.getApiToken())
                .build();

        try (Response response = httpClient.newCall(request).execute()) {
            if (!response.isSuccessful()) {
                throw new BusinessException("AI 调用失败: HTTP " + response.code());
            }
            // 从 OpenAI 兼容格式的响应中提取 content
            JsonNode root = objectMapper.readTree(response.body().string());
            return root.path("choices").get(0)
                       .path("message").path("content").asText();
        }
    }
}
```

### 8.2 AIService — 动态 Prompt 构建 & JSON 解析

```java
@Service
public class AIService {

    private final AIConfigMapper aiConfigMapper;
    private final AIClientFactory clientFactory;
    private final ObjectMapper objectMapper;

    private static final String SYSTEM_PROMPT_TEMPLATE = """
            你是一个专业的考试出题专家。请根据以下文档内容，生成一套高质量的题目。

            【题目要求】
            - 总题数：%d 道
            - 题型分布：
              单选题(SINGLE)：%d 道（%d%%）
              多选题(MULTI)：%d 道（%d%%）
              判断题(JUDGE)：%d 道（%d%%）
              %s

            【出题规范】
            1. 题目覆盖文档的核心知识点，避免重复考点
            2. 每道题标注难度：1-简单 2-中等 3-困难，三种难度均匀分布
            3. 每道题包含答案解析
            4. 单选题和多选题均需4个选项(A/B/C/D)，并标注正确答案
            5. 判断题选项固定为"正确"/"错误"
            6. 每道题默认分值 5 分
            7. 返回严格的 JSON 格式，不要包含任何其他文字
            ...
            """;

    public List<Question> generateQuestions(Long documentId, GenerateConfig config) {
        // 1. 查文档
        Document doc = documentMapper.selectById(documentId);
        // 2. 查激活的 AI 配置
        AIConfig aiConfig = aiConfigMapper.selectOne(
            new LambdaQueryWrapper<AIConfig>().eq(AIConfig::getIsActive, 1)
        );
        if (aiConfig == null) throw new BusinessException("未配置 AI 服务");

        // 3. 根据参数计算各题型数量
        int totalCount = config.getTotalCount();       // 20 / 50 / 60
        int singlePct  = config.getTypeRatios().getSingle();  // 50
        int multiPct   = config.getTypeRatios().getMulti();   // 30
        int judgePct   = config.getTypeRatios().getJudge();   // 20
        boolean includeFill = config.isIncludeFill();

        int singleCount = totalCount * singlePct / 100;
        int multiCount  = totalCount * multiPct  / 100;
        int judgeCount  = totalCount * judgePct  / 100;
        int fillCount   = 0;

        // 余数归入单选
        int remainder = totalCount - singleCount - multiCount - judgeCount;
        singleCount += remainder;

        String fillLine = "";
        if (includeFill) {
            // 填空题从各题型中等比例扣减10%
            fillCount = totalCount / 10;
            int deductEach = fillCount / 3;
            singleCount -= deductEach;
            multiCount  -= deductEach;
            judgeCount  -= (fillCount - deductEach * 2);
            fillLine = String.format("填空题(FILL)：%d 道", fillCount);
        }

        // 4. 构建动态 Prompt
        String systemPrompt = String.format(SYSTEM_PROMPT_TEMPLATE,
            totalCount,
            singleCount, singlePct,
            multiCount,  multiPct,
            judgeCount,  judgePct,
            fillLine
        );

        // 5. 调用 AI
        String aiResponse = clientFactory.callAI(aiConfig, systemPrompt, doc.getContentText());

        // 6. 解析 JSON，批量保存
        String json = extractJson(aiResponse);
        JsonNode root = objectMapper.readTree(json);
        List<Question> questions = new ArrayList<>();
        for (JsonNode qNode : root.path("questions")) {
            Question q = parseQuestionNode(qNode, documentId);
            questions.add(q);
        }
        questionService.batchSave(questions);

        // 7. 更新文档统计
        doc.setQuestionCount(questions.size());
        doc.setQuestionGenTime(LocalDateTime.now());
        documentMapper.updateById(doc);

        return questions;
    }

    private String extractJson(String raw) {
        raw = raw.trim();
        if (raw.startsWith("```")) {
            int start = raw.indexOf("\n") + 1;
            int end = raw.lastIndexOf("```");
            raw = raw.substring(start, end).trim();
        }
        return raw;
    }
}

## 十一、安全设计要点

1. **密码加密**：使用 `BCryptPasswordEncoder`，存储不可逆密文
2. **JWT Token**：包含 `userId`、`username`、`jti`（唯一 ID，用于登出黑名单），有效期 2h
3. **Token 刷新**：前端在 Axios 响应拦截器中处理 401，跳转登录页；可选实现 Refresh Token 机制
4. **权限控制**：
   - 后端：`@PreAuthorize("hasAuthority('doc:delete')")` 方法级注解
   - 前端：路由守卫检查 `meta.permission`，按钮使用 `v-if="hasPerm('doc:delete')"` 指令
5. **文件上传安全**：
   - 限制文件类型（仅允许 txt / docx / pdf）
   - 限制文件大小（建议最大 20MB）
   - 存储时重命名文件（UUID），防止路径遍历
6. **API Token 存储**：`ai_config.api_token` 字段建议使用 AES 加密存储，读取时解密

### 12.3 Nginx 核心配置

```nginx
server {
    listen 80;
    server_name quiz.example.com;

    # 前端
    location / {
        root /opt/ai-quiz-web/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 上传文件
    location /uploads/ {
        alias /data/uploads/;
    }
}
```
